Linux の /bin ディレクトリは、システムの基本的な動作に必要な基本的なバイナリファイルが格納されている場所です。このディレクトリに含まれるコマンドは、システムが正常に動作するために不可欠で、通常、ルートユーザーだけでなく一般ユーザーも使用可能なコマンドが多くあります。

---

1. cat

用途: ファイルの内容を表示、連結、標準入力から標準出力への出力を行う。

例: cat file.txt → ファイルの内容を表示。


2. chgrp

用途: ファイルやディレクトリのグループ所有権を変更する。

例: chgrp users file.txt → ファイルのグループを "users" に変更。


3. chmod

用途: ファイルやディレクトリの権限を変更する。

例: chmod 755 script.sh → スクリプトに実行権を付与。


4. chown

用途: ファイルやディレクトリの所有者を変更する。

例: chown user file.txt → ファイルの所有者を "user" に変更。


5. cp

用途: ファイルやディレクトリをコピーする。

例: cp file.txt backup.txt → ファイルをコピー。


6. date

用途: 現在の日付と時刻を表示または設定する。

例: date → 現在の日付を表示。


7. dd

用途: データを指定した形式でコピーする（ディスク操作に使用）。

例: dd if=/dev/sda of=backup.img → ディスクイメージの作成。


8. df

用途: ファイルシステムの使用量を表示する。

例: df -h → 人間が読みやすい形式で使用量を表示。


9. dmesg

用途: システムブート時やカーネルメッセージを表示。

例: dmesg | grep error → エラーメッセージを検索。


10. echo

用途: 引数を標準出力に出力。

例: echo "Hello, World!" → メッセージを表示。


11. false

用途: 常に失敗するステータスコードを返す。

例: スクリプト内で条件分岐として使用。


12. hostname

用途: システムのホスト名を表示または設定する。

例: hostname → 現在のホスト名を表示。


13. kill

用途: プロセスにシグナルを送る。

例: kill 1234 → プロセスID 1234 を終了。


14. ln

用途: ハードリンクまたはシンボリックリンクを作成する。

例: ln -s file.txt link.txt → シンボリックリンクを作成。


15. login

用途: システムにログインする。

例: 通常システム内部で使用。


16. ls

用途: ディレクトリの内容を表示。

例: ls -l → 詳細情報付きでリストを表示。


17. mkdir

用途: 新しいディレクトリを作成。

例: mkdir new_folder → ディレクトリを作成。


18. mknod

用途: 特殊ファイル（デバイスファイルなど）を作成する。

例: 管理者用途。


19. more

用途: テキストファイルを1画面ずつ表示。

例: more file.txt → 画面単位で表示。


20. mount

用途: デバイスをマウント（接続）する。

例: mount /dev/sdb1 /mnt → デバイスを /mnt にマウント。


21. mv

用途: ファイルやディレクトリを移動または名前変更。

例: mv oldname.txt newname.txt → ファイル名を変更。


22. ps

用途: 実行中のプロセスを表示。

例: ps aux → 全プロセスを詳細表示。


23. pwd

用途: 現在の作業ディレクトリを表示。

例: pwd → 現在のディレクトリを表示。


24. rm

用途: ファイルやディレクトリを削除。

例: rm file.txt → ファイルを削除。


25. rmdir

用途: 空のディレクトリを削除。

例: rmdir empty_folder → 空のディレクトリを削除。


26. sed

用途: テキストストリームを編集。

例: sed 's/old/new/g' file.txt → 文字列を置換。


27. sh

用途: シェルスクリプトを実行。

例: sh script.sh → スクリプトを実行。


28. stty

用途: ターミナルの設定を変更。

例: 通常システム管理者が使用。


29. su

用途: 別のユーザーに切り替える。

例: su root → 管理者ユーザーに切り替え。


30. sync

用途: ディスクへの変更を即座に書き込む。

例: sync → キャッシュされたデータをディスクに書き込む。


31. true

用途: 常に成功するステータスコードを返す。

例: スクリプト内で条件分岐として使用。


32. umount

用途: デバイスのマウントを解除。

例: umount /mnt → /mnt のマウント解除。


33. uname

用途: システム情報を表示。

例: uname -a → システムの詳細情報を表示。



---

/binディレクトリの特徴

1. システムの基本操作を担う:
多くのコマンドは、システムの初期化時や基本的なタスク（ファイル管理、システム情報の取得）に使用されます。


2. 他のディレクトリとの違い:
/bin はシングルユーザーモード（最小限の起動状態）でも使用可能なコマンドが入っています。一方、/usr/bin は追加のアプリケーションやツールが含まれます。


3. POSIX準拠:
ls や cp などのコマンドは、POSIX標準で定義されており、ほぼすべてのUnix系システムで使用可能です。
